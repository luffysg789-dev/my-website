# SBTI 设计文档

**目标**

新增一个 `SBTI` 测试游戏，支持 PC 和手机端，收录到游戏大全。玩法参考 [sbti.unun.dev](https://sbti.unun.dev/)，首版实现欢迎页、连续答题页、结果页三段式流程，不接登录、不接支付、不接后端存档。

**范围**

- 新增独立页面路由 `/sbti/`
- 游戏大全展示 `SBTI`
- 欢迎页展示标题、简介、开始测试按钮
- 答题页展示固定题库、进度条、作答状态、提交按钮
- 结果页展示类型、描述、维度分数、友情提示、重新测试
- PC/手机自适应

**不做**

- 不做账号登录
- 不做服务端记录
- 不做分享海报
- 不做多语言

## 交互流程

### 1. 欢迎页

用户打开页面先看到简介卡片和开始按钮。点击 `开始测试` 后进入答题页。

### 2. 答题页

- 固定 `31` 题
- 每题单选 `5` 档
- 必须答完全部题目才能提交
- 顶部显示进度条和 `已完成题数 / 总题数`
- 底部显示提醒文案和提交按钮

### 3. 结果页

提交后根据四个维度得分计算类型，展示：

- 类型代码
- 中文类型名
- 一段结果描述
- 四个维度的分数与倾向
- 一段友情提示
- `重新测试`

## 数据模型

前端本地维护：

- `questions`: 题库数组
- `answers`: `{ [questionId]: optionValue }`
- `dimensions`: 4 个维度定义
- `types`: 16 种类型映射

不持久化用户结果，只保留当前页面会话内状态。

## 结果计算

采用 4 轴 16 型：

- `S / C`
- `B / A`
- `T / F`
- `I / O`

每题映射到某个维度，选项值为 `1-5`。根据维度累计分数，取高低侧拼出类型代码，例如：

- `SBTI`
- `CBFO`

再映射到结果名称与描述。

## 视觉设计

延续参考站的卡片式测试气质，但不逐字复刻：

- 浅色背景
- 圆角卡片
- 绿色主色系
- 结果页强调类型名和分数块

移动端要求：

- 单列布局
- 按钮足够高
- 文案行高舒适
- 提交按钮固定在答题流下方，不遮挡内容

## 文件结构

- `public/sbti/index.html`
  页面结构和挂载点
- `public/sbti/style.css`
  样式和响应式布局
- `public/sbti/script.js`
  题库、计算逻辑、页面切换
- `public/games-config.js`
  游戏大全配置
- `src/db.js`
  默认游戏目录补充
- `src/server.js`
  路由映射补充
- `tests/sbti-page.test.js`
  页面接入和结构测试
- `tests/games-page-layout.test.js`
  游戏大全接入断言

## 测试策略

- 测试页面文件存在
- 测试游戏大全已接入
- 测试后端默认目录已接入
- 测试 HTML 包含欢迎页、答题页、结果页挂载点
- 测试脚本包含题库、进度、结果计算和重测逻辑
- 测试 CSS 包含响应式断点和卡片布局
